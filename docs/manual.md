# agentsync ŌĆö User Manual

A shared, repo-local memory layer that synchronizes context across AI coding agents. Edit one file in `agent/`; every supported tool's context (`CLAUDE.md`, `AGENTS.md`, `.cursorrules`, `.clinerules`, `.windsurfrules`, `.github/copilot-instructions.md`) is regenerated by a single command.

Deterministic. Local-first. LLM-free in the default path.

## The problem

Most teams now use more than one AI coding agent. Claude Code, Codex, Cursor, Cline, Windsurf, and Copilot all read context from a tool-specific file at the root of your repo:

| Tool | Reads |
|---|---|
| Claude Code | `CLAUDE.md` |
| Codex / OpenAI | `AGENTS.md` |
| Cursor | `.cursorrules` (+ `.cursor/rules/*.mdc`) |
| Cline | `.clinerules` |
| Windsurf | `.windsurfrules` |
| GitHub Copilot | `.github/copilot-instructions.md` |

Today, each of those files is hand-maintained. As soon as your repo's conventions change, all six files drift apart, and you re-explain your project to every agent over and over.

`agentsync` flips the model: you maintain a single `agent/` directory with your project's architecture, coding rules, and workflows. One command regenerates all six per-tool files deterministically.

## Quickstart

A 60-second walk-through. Run inside any git repo.

```bash
agentsync init                # scaffold agent/ with starter templates
$EDITOR agent/coding-rules.md # describe your conventions
agentsync sync                # regenerate CLAUDE.md, AGENTS.md, and 5 others
agentsync scan                # generate agent/stack.md + agent/repo-map.json
agentsync lint                # confirm memory files are well-formed
```

After this, your repo has:

```
agent/
Ōö£ŌöĆŌöĆ manifest.yaml
Ōö£ŌöĆŌöĆ architecture.md
Ōö£ŌöĆŌöĆ coding-rules.md
Ōö£ŌöĆŌöĆ domain-knowledge.md
Ōö£ŌöĆŌöĆ stack.md         (generated by scan)
Ōö£ŌöĆŌöĆ repo-map.json    (generated by scan)
ŌööŌöĆŌöĆ workflows/
    Ōö£ŌöĆŌöĆ testing.md
    Ōö£ŌöĆŌöĆ deployment.md
    ŌööŌöĆŌöĆ debugging.md
CLAUDE.md                          (generated by sync)
AGENTS.md                          (generated by sync)

.cursor/rules/agentsync.mdc        (generated by sync)
.clinerules                        (generated by sync)
.windsurfrules                     (generated by sync)
.github/copilot-instructions.md    (generated by sync)
```

Edit anything in `agent/` and re-run `agentsync sync`. The six per-tool files update in one shot.

## Concepts

### `agent/` ŌĆö the source of truth

Every memory file lives under `agent/`. Files are markdown with a YAML frontmatter block. `agent/` is checked into version control; PR review of `agent/coding-rules.md` is now PR review of your team's AI conventions.

### `agent/manifest.yaml`

Declares the project name, the active adapter targets, and every file under `agent/` with three pieces of metadata:

```yaml
version: 1
project:
  name: my-app
targets:
  - claude
  - agents-md
  - cursor
files:
  - path: architecture.md
    source: hybrid
    priority: 10
    applies_to: ["*"]
  - path: coding-rules.md
    source: authored
    priority: 20
    applies_to: ["*"]
```

Three concepts inside the manifest:

- **`source`** ŌĆö `authored` (human-written; `sync` never touches), `generated` (`scan` overwrites; e.g. `stack.md`), or `hybrid` (mixed ŌĆö generated blocks delimited by HTML comments inside human prose).
- **`priority`** ŌĆö lower numbers = higher placement when assembling per-tool output. Used to decide ordering and to drop the right sections first under token budget pressure.
- **`applies_to`** ŌĆö array of adapter names. `["*"]` means "every adapter"; `["claude", "cursor"]` means only those two.

### Adapters

An adapter is a pure function: `(MemorySet, RenderContext) ŌåÆ GeneratedFile[]`. agentsync ships six built-in adapters; you can write more via the `@agentsync/adapter-sdk` package. See [Writing custom adapters](#writing-custom-adapters).

### Drift detection

agentsync remembers what it wrote, where, and from what source ŌĆö stored in `.agentsync/last-sync.json`. On the next `sync`, it three-way-compares:

- **A** = current `agent/` source SHA
- **B** = last recorded content checksum
- **C** = current on-disk file checksum

If `B == C`, the generated file is unchanged since last sync ŌåÆ safe to overwrite.
If `B != C`, somebody hand-edited the generated file ŌåÆ `sync` refuses to overwrite without `--force`, and `sync --check` exits with code 2.

agentsync never tries to merge your edits back into `agent/`. If you edited `CLAUDE.md` directly: copy your changes back into the relevant memory file under `agent/` and re-run `sync`.

### Provenance headers

Every generated file starts with an HTML-comment block:

```
<!-- agentsync:generated -->
<!-- source-sha: abc123... -->
<!-- generator: claude-adapter@0.1.0 -->
<!-- generated-at: 2026-05-31T12:34:56.789Z -->
<!-- DO NOT EDIT ŌĆö run `agentsync sync` instead -->
```

The header is stripped before checksumming, so updating the timestamp on a fresh `sync` doesn't trigger drift.

### Determinism contract

Same input ŌåÆ byte-identical output across runs, machines, and OSes. agentsync enforces this by:

- Writing every output with LF line endings via a single `writeFileLF` helper.
- Sorting all lists with a locale-independent comparator (never `localeCompare`).
- Excluding per-run timestamps from comparisons (provenance stripping).
- Skipping its own outputs during scan to avoid feedback loops.

If `agentsync sync` produces different bytes on two clean runs, that's a bug ŌĆö please file an issue.

## Commands

Every command supports these global flags:

| Flag | Purpose |
|---|---|
| `--cwd <path>` | Run as if from this directory |
| `--quiet` | Suppress non-error output |
| `--verbose` | Verbose diagnostics |
| `--no-color` | Disable ANSI color (also auto-disabled when piped) |
| `--json` (per command) | Machine-readable output on stdout |

Exit codes are stable:

| Code | Meaning |
|---|---|
| 0 | OK |
| 1 | User error (bad input, missing file, etc.) |
| 2 | Drift detected (CI signal) |
| 3 | Internal error (a bug ŌĆö please report) |

Logs go to stderr. Structured (`--json`) output goes to stdout.

---

### `agentsync init`

Scaffold a fresh `agent/` directory using a bundled template.

```bash
agentsync init [--template <name>] [--targets claude,cursor,...] [--yes]
```

| Flag | Purpose |
|---|---|
| `--template <name>` | Template name (default: `starter`) |
| `--targets <list>` | Comma-separated adapter targets to enable in the new manifest |
| `--yes` | Allow merge into an existing `agent/` instead of erroring |

The template's `{{projectName}}` is substituted from your `package.json` (`name` field, scope stripped) or falls back to the directory basename.

---

### `agentsync validate`

Validate every memory file's frontmatter, every manifest entry, and every intra-repo link.

```bash
agentsync validate [--strict] [--json]
```

- Errors: missing frontmatter, schema violations, files declared in manifest but missing on disk (non-generated only).
- Warnings: broken markdown links, files on disk not declared in manifest, generated files missing on disk.
- `--strict` escalates warnings to errors.

---

### `agentsync show <file>`

Pretty-print a memory file with parsed frontmatter and body, or emit it as JSON.

```bash
agentsync show <file> [--json]
```

You can pass an absolute path, a path relative to `cwd`, or a path relative to `agent/` (agentsync looks in all three).

---

### `agentsync sync`

Regenerate per-tool output files from `agent/`.

```bash
agentsync sync [--check] [--adapter <names...>] [--dry-run] [--force] [--json]
```

| Flag | Purpose |
|---|---|
| `--check` | Exit code 2 if any output file would change. CI mode. |
| `--adapter <names>` | Limit to specific adapters (one or more). |
| `--dry-run` | Show what would be written without writing. |
| `--force` | Overwrite files even if they show signs of manual editing. |
| `--json` | Machine-readable summary. |

`sync` is idempotent: a second run after a clean first run is a no-op. If you run it twice and the second run reports changes, that's a bug.

---

### `agentsync diff [adapter]`

Preview what `agentsync sync` would change, with a per-file unified-diff view.

```bash
agentsync diff [<adapter>] [--json]
```

If `<adapter>` is omitted, diffs every active adapter. The output classifies each file as `new`, `up-to-date`, `would-update`, or `drifted (manually edited)`.

---

### `agentsync scan`

Walk the repo and (re)generate `agent/repo-map.json` and `agent/stack.md`.

```bash
agentsync scan [--check] [--no-cache] [--incremental] [--json]
```

| Flag | Purpose |
|---|---|
| `--check` | Exit code 2 if scan artifacts would change. CI mode. |
| `--no-cache` | Ignore `.agentsync/cache/scan.json` for this run. |
| `--incremental` | Reserved for a future watch mode. Currently a no-op alias. |

Scan deliberately does no AST parsing in Phase 1 ŌĆö it's a filesystem walk + a line counter + a stack signature reader. Output:

- **`agent/repo-map.json`** ŌĆö hierarchical tree of the repo with per-file `language`, `role` (entry / test / config / doc / source), and `lineCount`. Top-level summary lists per-language line totals.
- **`agent/stack.md`** ŌĆö human-readable summary auto-detected from `package.json`, `tsconfig.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `Dockerfile`, CI configs, and JS/Python framework signatures.

---

### `agentsync lint`

Lint memory files against five built-in rules:

| Rule | Default | Detects |
|---|---|---|
| `heading-hierarchy` | warn | Skips in heading levels (`#` ŌåÆ `###`) |
| `banned-vague-phrases` | warn | "TODO", "TBD", "lorem ipsum", "fill in", etc. |
| `required-sections` | error | Missing sections declared in the policy |
| `freshness` | warn | File mtime older than `maxAgeDays` (default 90) |
| `max-length` | warn | Body exceeds `maxBodyChars` (default 20000) |

```bash
agentsync lint [--fix] [--policy <path>] [--json]
```

`--fix` is reserved for rules that can safely auto-fix; current built-ins are advisory.

### `agentsync stats`

Report per-file character/token counts, per-adapter token counts, and current drift status.

```bash
agentsync stats [--json]
```

Use the `--json` shape in scripts:

```bash
agentsync stats --json | jq '.adapters[] | select(.driftStatus == "drifted")'
```

### `agentsync export <path>`

Bundle the current `agent/` directory into a portable `.tar.gz`.

```bash
agentsync export <path> [--json]
```

Honors `.agentsyncignore`. The resulting tarball is suitable for sharing as a starter template or moving an entire memory set between repos.

---

### `agentsync import <source>`

Pull a starter `agent/` from a local directory, a `.tar.gz`, or a git URL.

```bash
agentsync import <source> [--into <path>] [--branch <ref>] [--force] [--json]
```

| Flag | Purpose |
|---|---|
| `--into <path>` | Destination (default: `agent/`) |
| `--branch <ref>` | Git ref to clone (git sources only) |
| `--force` | Overwrite an existing non-empty destination |

Examples:

```bash
agentsync import ./templates/nextjs           # local directory
agentsync import bundle.tar.gz                # local tarball
agentsync import https://github.com/me/agent  # git URL
agentsync import git@github.com:me/t.git --branch v2 --into agent
```

### `agentsync install-hook [type]`

Install a git pre-commit hook that runs `agentsync sync --check`.

```bash
agentsync install-hook              # native git hook (default)
agentsync install-hook husky        # Husky hook (requires npx husky init first)
```

The native hook writes to `.git/hooks/pre-commit` and is `chmod +x`.

## Configuration files

### `.agentsync/config.yaml`

Per-repo configuration. Created automatically by `agentsync init`. All fields are optional:

```yaml
defaultAdapters:
  - claude
  - agents-md
tokenBudgets:
  "*": 8000        # global budget
  claude: 10000    # per-adapter override
languageMap:
  nim: Nim         # extend the scan's built-in language map
plugins:
  adapters: []
  scanners: []
  lintRules: []
```

### `agent/.agentsyncignore`

`gitignore`-style file. Anything matched is excluded from `loadMemorySet` and from `agentsync export`.

### `.agentsync/last-sync.json`

Drift-detection state. Should **not** be committed (add to `.gitignore`). Regenerated by every `sync` run; safe to delete.

### `.agentsync/cache/`

Content-hash cache used by `agentsync scan` to skip work on files that haven't changed. Safe to delete. Already gitignored by default.

## Team policy

For teams that want to enforce conventions across repos, drop `agentsync.policy.yaml` at the repo root (or pass `--policy <path>`):

```yaml
version: 1

# Block sync if any of these files are missing
requiredFiles:
  - architecture.md
  - coding-rules.md

# Block sync if any of these adapters aren't in the manifest's targets
requiredAdapters:
  - claude
  - agents-md

lint:
  rules:
    banned-vague-phrases:
      level: error              # promote from default warn to error
      extra: ["WIP", "draft"]   # additional banned phrases
    freshness:
      level: warn
      maxAgeDays: 60            # default is 90
    max-length:
      maxBodyChars: 30000
  requiredSections:
    architecture.md:
      - "## System overview"
      - "## Modules"

tokenBudgets:
  claude: 8000
  cursor: 4000
```

Three commands accept `--policy <path>`: `sync`, `validate`, `lint`. The policy file is loaded once per command invocation and merged with `.agentsync/config.yaml`.

## CI integration (GitHub Action)

Drop this file at `.github/workflows/agentsync.yml`:

```yaml
name: agentsync

on:
  pull_request:
    paths:
      - "agent/**"
      - "CLAUDE.md"
      - "AGENTS.md"
      - ".cursorrules"
      - ".cursor/rules/**"
      - ".clinerules"
      - ".windsurfrules"
      - ".github/copilot-instructions.md"

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npx -y @agentsync/cli sync --check
      - run: npx -y @agentsync/cli scan --check
      - run: npx -y @agentsync/cli lint
```

When `@agentsync/cli` ships on the npm registry, you can also use the bundled composite action from `packages/action/`:

```yaml
- uses: tasnuvaleeya/memory-synchronizer/packages/action@v1
  with:
    command: "sync --check,lint"
    policy: agentsync.policy.yaml
```

The action surfaces lint findings as inline PR annotations.

## MCP server

`agentsync mcp` exposes your `agent/` directory as Model Context Protocol resources, so any MCP-aware client (Claude Desktop, Claude Code, custom agents) can list and read your memory files programmatically.

Full wiring guide: [docs/mcp.md](./mcp.md).

Resources exposed (all read-only):

| URI | MIME |
|---|---|
| `agentsync://manifest` | `application/yaml` |
| `agentsync://memory/list` | `application/json` |
| `agentsync://memory/<path>` | `text/markdown` |
| `agentsync://adapters/<name>` | `text/markdown` |
| `agentsync://scan/repo-map` | `application/json` |
| `agentsync://scan/stack` | `text/markdown` |

## Writing custom adapters

Install the SDK:

```bash
npm install @agentsync/adapter-sdk
```

Define an adapter:

```ts
import {
  type Adapter,
  type GeneratedFile,
  applyTokenBudget,
  computeSourceSha,
  injectProvenance,
  contentChecksum,
} from "@agentsync/adapter-sdk";

const myAdapter: Adapter = {
  name: "my-tool",
  version: "0.1.0",
  outputPaths: [".mytoolrc"],

  async render(memory, ctx): Promise<GeneratedFile[]> {
    const { sections } = applyTokenBudget(memory, ctx, "my-tool");
    const body = sections.map((s) => s.body).join("\n\n");
    const sourceSha = computeSourceSha(memory);
    const contents = injectProvenance(body, {
      sourceSha,
      generator: `my-tool-adapter@0.1.0`,
      generatedAt: ctx.now,
    });
    return [{
      path: ".mytoolrc",
      contents,
      checksum: contentChecksum(contents),
      sourceSha,
    }];
  },
};

export default myAdapter;
```

Register it in `.agentsync/config.yaml`:

```yaml
plugins:
  adapters:
    - "@my-org/agentsync-adapter-mytool"
```

Full SDK reference: [packages/adapter-sdk/README.md](../packages/adapter-sdk/README.md).

## Troubleshooting

### "Drift detected" on a file I didn't touch

Most often this is editor-induced: a save with CRLF line endings, a trailing-whitespace strip, or an extra blank line at end-of-file. Re-save the file with LF endings (most editors expose this in the status bar). If that doesn't help, `agentsync diff <adapter>` will show the byte-level diff.

### "No `agent/` directory found"

You're running from above the repo root, or `agent/` was renamed/deleted. Pass `--cwd <repo-root>` or `cd` to the right place.

### `agentsync scan` shows a wrong language for a file

The default language map covers the most common extensions. Add overrides in `.agentsync/config.yaml`:

```yaml
languageMap:
  nim: Nim
  zig: Zig
```

### `agentsync mcp` "isn't responding" when I run it in a terminal

`agentsync mcp` is meant to be launched by an MCP client, not standalone. It's waiting on stdin for JSON-RPC. To smoke-test:

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"resources/list"}' | agentsync mcp
```

### My `sync && sync` reports changes on the second run

That's a determinism bug. Please file an issue with:
- OS + Node version
- `agentsync version` output
- The exact diff (`agentsync diff <adapter>`)

### CI's `sync --check` fails but local `sync` is clean

Common cause: Windows runner with CRLF line endings, or a `.gitattributes` rule that normalizes differently than `agentsync`'s `writeFileLF`. Configure your CI runner to check out files with LF line endings (`git config --global core.autocrlf input`).

### Memory file lint warns about freshness even though I just edited it

`freshness` uses the on-disk mtime, not git history. `git checkout` and CI checkouts reset mtimes to the checkout time. The rule is best-suited to local development; consider disabling it in `agentsync.policy.yaml` for CI:

```yaml
lint:
  rules:
    freshness:
      level: off
```

## FAQ

**Q: Why isn't this an LLM-powered tool?**
A: Determinism is the brand. Same input ŌåÆ same output. No API keys, no flakiness, no rate limits. We will likely add an *opt-in* `@agentsync/ai-draft` companion package later (drafting `architecture.md` from scan output), but it will be a separate install ŌĆö the core CLI stays LLM-free.

**Q: Why six built-in adapters specifically?**
A: They're the six tools we've seen real teams actively using as of early 2026. The plugin system means any vendor or community can ship a 50-line adapter for any new tool ŌĆö `pnpm install @agentsync/adapter-<name>`, list it under `plugins.adapters` in `.agentsync/config.yaml`, done.

**Q: Can I sync only some adapters?**
A: Yes ŌĆö `agentsync sync --adapter claude --adapter cursor`. Or remove the adapter from your manifest's `targets`.

**Q: Does this work in a monorepo?**
A: Yes. Run `agentsync init` at each package root, or at the monorepo root. Each `agent/` is independent.

**Q: How do I version my agent memory?**
A: Check `agent/` into git. PR review of `agent/coding-rules.md` is now PR review of your team's AI conventions. The optional GitHub Action enforces consistency on every PR.

**Q: What about secrets?**
A: agentsync does not read source code in Phase 1 ŌĆö only filesystem shape and stack signatures. There's no path for source content (and therefore secrets) to leak into generated artifacts today. When AST scanning lands in Phase 2 (M3.5), a regex-based secret redactor will scrub any source-derived content before it's written.

**Q: Where is the spec?**
A: JSON Schemas for `manifest.yaml`, `Frontmatter`, `Config`, `RepoMap`, `Stack`, `MemoryFile`, and `Policy` are exported under [`docs/schema/v1/`](./schema/v1/).
